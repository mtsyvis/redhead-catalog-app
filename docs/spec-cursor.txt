REDHEAD – (SITES CATALOG) – CURSOR IMPLEMENTATION SPEC
Target model for Cursor: Claude Sonnet 4.5
Architecture: Single ASP.NET Core app serves API + React SPA

============================================================
1) Goal
============================================================
Build a production-ready internal + client web app to browse/export a Sites catalog (~60k rows), with:
- Email+password authentication (no self-registration)
- Role-based access control (RBAC)
- Admin-managed per-role CSV export limits
- Fast server-side search / filters / paging / sorting
- Multi-search by pasted list of domains/URLs (found + not found)
- Imports:
  - MVP: CSV only. Sites import (ADD-ONLY) from CSV; skip duplicates; report
  - Quarantine import (domain + reason) from CSV; mark sites “Unavailable”
  - XLSX import is Future Scope (optional)
- Basic audit logs for exports/imports

MVP explicitly does NOT include:
- Bulk price update by import (upsert)
- Global mass price adjustment tools (multiplier/additive)
These are “Future Scope” (separate scope later).

============================================================
3) Roles & Permissions (RBAC)
============================================================
One role per user.

Roles:
- SuperAdmin
  - Full control over everything
  - Can reset passwords for Admins
  - Can remove/disable any account (including Admins)
  - Can manage RoleSettings, imports, edit sites, export, view logs
- Admin
  - Can manage users (including creating other Admins)
  - Can manage RoleSettings
  - Can run imports (Sites + Quarantine)
  - Can edit sites
  - Can export (subject to RoleSettings limit)
- Internal
  - Read-only sites + export
  - Export limit applies (configured in RoleSettings)
- Client
  - Read-only sites + export
  - Export limit applies (configured in RoleSettings; different limit than Internal)

Authorization:
- All API routes require auth by default
- Enforce RBAC server-side via policies, not via UI hiding

User Soft Delete (IsActive flag):
- Users have an `IsActive` boolean field (default: true)
- "Delete user" in UI sets `IsActive = false` (soft delete)
- When `IsActive = false`:
  - User cannot login
  - User cannot access any API endpoint
- Physical deletion is NOT allowed (audit logs must preserve user references)

============================================================
4) Password provisioning (generated + must-change-on-first-login)
============================================================
No self-registration.

Create user flow:
- Admin creates user (email + role)
- System generates temporary random password
- User MUST change password at first login:
  - Persist `MustChangePassword` boolean on user (default: true for new users)
  - After login, if MustChangePassword==true, UI forces /change-password and blocks everything else

Reset password:
- SuperAdmin can reset passwords for Admins and any other users
- Admin can reset passwords for non-Admin users:
  - Generate new temp password
  - Set MustChangePassword=true

Security notes (keep minimal, but correct):
- Temp password displayed ONCE in UI after creation/reset
- Do not store temp password in DB logs
- Rate limit login attempts (optional MVP)

User entity fields (to be implemented in Identity commit):
- IsActive (bool, default: true) – soft delete flag
- MustChangePassword (bool, default: true for new users) – force password change

5) Data Model
============================================================
5.1 Sites (from Sites.xlsx)
Fields:
- Id (GUID or long)
- Domain (normalized host, unique)
- DR (number)
- Traffic (number)              // from “Traf 11/12/25”
- Location (string)
- PriceUsd (decimal)
- PriceCasino (decimal? nullable)     // null => NOT allowed
- PriceCrypto (decimal? nullable)     // null => NOT allowed
- PriceLinkInsert (decimal? nullable) // null => NOT allowed
- Niche (string, nullable)            // stored, no filtering in MVP
- Categories (string, nullable)       // stored, no filtering in MVP
- IsQuarantined (bool)
- QuarantineReason (string, nullable)
- QuarantineUpdatedAtUtc (datetime?, nullable)
- CreatedAtUtc, UpdatedAtUtc

Indexes:
- Unique on Domain
- Index on Location
- Index on IsQuarantined
- (Optional) indexes on DR, Traffic, PriceUsd

5.2 RoleSettings
- RoleName (PK)
- ExportLimitRows (int)
Rules:
- ExportLimitRows == 0 => export disabled
- “Unlimited” = big number (e.g., 1000000) configured per role

Default limits per role:
- SuperAdmin: 1000000 (unlimited)
- Admin: 1000000 (unlimited)
- Internal: 10000
- Client: 5000

5.3 Audit tables (minimum viable)
- ExportLog:
  - Id, UserId, UserEmail, Role, TimestampUtc, RowsReturned, FilterSummaryJson(optional)
- ImportLog:
  - Id, UserId, UserEmail, Type(Sites/Quarantine), TimestampUtc,
    Inserted, Duplicates, Matched, Unmatched, ErrorsCount

============================================================
6) Domain Normalization (must be consistent everywhere)
============================================================
Normalize and store/match by domain host only.

Normalization rules:
- Trim whitespace
- Remove scheme: http://, https://
- Remove leading www.
- Remove trailing slash
- If input contains path/query/fragment, keep only host
- Lowercase

Example:
  https://www.Example.com/path?q=1  -> example.com

Use normalization in:
- Sites import
- Quarantine import
- Single search
- Multi-search

9) Imports
============================================================
9.1 Sites import (ADD-ONLY)
Input: CSV only (MVP). Same columns as existing file. XLSX is Future Scope.

Rules:
- Normalize domain for each row
- If Domain already exists => skip, record in duplicates list
- Else insert
- Parsing/validation errors are collected and shown; do not crash the entire import

Output:
- Inserted count
- Duplicates count + list
- Errors count + list (row number + message)

Performance:
- Batch inserts (e.g., 1000 rows)
- Avoid loading all 60k into memory if possible

Log:
- Insert ImportLog row

9.2 Quarantine import (domain + reason)
Input: CSV only (MVP). XLSX is Future Scope. Columns:
- Column A: domain/url
- Column B: quarantine reason (string)

Rules:
- Normalize domain
- If exists:
  - IsQuarantined = true
  - QuarantineReason = reason (trim/sanitize)
  - QuarantineUpdatedAtUtc = now
- If not found: add to unmatched list

Output:
- matched count
- unmatched list
- errors list

Log:
- Insert ImportLog row

11) API Endpoints (suggested)
============================================================
Auth
- POST /api/auth/login
- POST /api/auth/logout
- GET  /api/auth/me
- POST /api/auth/change-password

Sites
- GET  /api/sites
  - query: page, pageSize, sortBy, sortDir, q,
           drMin/drMax, trafficMin/trafficMax, priceMin/priceMax,
           locations[], casinoAllowed, cryptoAllowed, linkInsertAllowed,
           quarantine=all|only|exclude
- POST /api/sites/multi-search  { inputs: string[] }
- PUT  /api/sites/{id}          (SuperAdmin/Admin)

Export
- GET /api/export/sites.csv (same query params as /api/sites)

Imports
- POST /api/import/sites      (SuperAdmin/Admin) multipart file
- POST /api/import/quarantine (SuperAdmin/Admin) multipart file

Users
- POST /api/users                      (SuperAdmin/Admin)
- GET  /api/users                      (SuperAdmin/Admin)
- DELETE /api/users/{id}               (SuperAdmin/Admin) - soft delete (sets IsActive=false)
- POST /api/users/{id}/reset-password  (SuperAdmin can reset any; Admin can reset non-Admins)

Role Settings
- GET /api/role-settings (SuperAdmin/Admin)
- PUT /api/role-settings (SuperAdmin/Admin)

Logs (optional UI; endpoints useful)
- GET /api/logs/exports (Admin)
- GET /api/logs/imports (Admin)
